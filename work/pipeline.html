Skill Composer — 查询特定技能合成流水线

问题背景

冗余信息多
原始技能文档为通用场景设计，包含大量与当前任务无关的内容
冲突与重复
多技能组合时存在重复指令和相互冲突的操作步骤
堆叠后过于冗长
直接拼接多个技能文档，导致上下文过长、噪声大
Extract 抽取
解析 SKILL.md
提取结构化字段：核心步骤、约束条件、示例、依赖项
字段标准化
统一不同技能作者的格式差异
Select 筛选
逐项相关性判断
LLM 逐条判断每个字段与任务查询的相关性
硬过滤
丢弃 keep=false 的项，保留其余不变
Resolve 消解
去重合并
合并多个候选技能中相同或近似的指令
冲突解决
任务优先策略：保留与任务更相关的版本
依赖补全
补充被保留项所隐含的依赖项
Compress 压缩
紧凑渲染
以目标压缩比渲染最终 SKILL.md
可读性检查
确保输出连贯、自包含、可执行
输入
N × SKILL.md + 任务查询
→
输出
精简 SKILL.md（任务专用）
